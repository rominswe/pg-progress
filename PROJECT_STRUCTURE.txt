================================================================================
2. DETAILED DIRECTORY DESCRIPTIONS
================================================================================

## Root Directory (/)
- .env: Environment variables configuration.
- .gitignore: Specifies intentionally untracked files to ignore.
- docker-compose.yaml: Docker orchestration configuration for running services (db, redis, server, client, etc.).
- LICENSE: License information.
- README.md: Project documentation.
- package.json: Root level scripts and dependencies.
- package-lock.json: Dependency tree lock file.

## Client Application (/client)
Built with Vite and React.

### Configuration Files
- admin.html: Entry point for the Admin application.
- user.html: Entry point for the User application.
- vite.config.js: Vite bundler configuration.
- tailwind.config.js: Tailwind CSS configuration.
- postcss.config.js: PostCSS configuration.
- jsconfig.json: JavaScript path aliasing configuration.
- eslint.config.js: Linting rules.

### Source Code (/client/src)
- admin/: Admin-specific application logic.
  - App.jsx, main.jsx, index.css: Entry files for admin portal.
- user/: User-specific application logic.
  - App.jsx, main.jsx, index.css: Entry files for user portal.
- components/: Reusable React components.
  - auth/: Authentication forms and context (Login, ProtectedRoute, AuthContext).
  - form/: Detailed business forms.
    - examiner/: Examiner specific forms (ThesisEvaluationForm.jsx).
    - StudentForm, SupervisorAssessmentForm, etc.
  - layout/: Page layouts (Layout, StaffLayout, StudentLayout, SupervisorLayout).
  - modal/: Modal dialogs (ConfirmRegisterModal).
  - ui/: Shadcn/UI primitive components (button, card, input, etc.).
- hooks/: Custom React hooks.
- pages/: Application page views.
  - examiner/: Pages for Examiners.
    - ExaminerDashboard.jsx: Dashboard for examiners to view assigned students.
  - staff/: Pages for CGS/Staff users (Dashboard, RegisterUsers, VerifyDocuments, etc.).
  - student/: Pages for Student users (Dashboard, ThesisSubmission, Uploads, ProgressUpdates).
  - supervisor/: Pages for Supervisors (Dashboard, ReviewSubmissions, ProgressEvaluation).
- services/: API integration services.
- lib/: Utility libraries.

## Server Application (/server)
Built with Node.js and Express.

### Configuration & Root Files
- app.js: Express application instance setup.
- server.js: Server entry point (starts listening on ports).
- Dockerfile: Docker build instructions for the server.
- package.json: Server-specific dependencies.
- uploads/: Directory for storing uploaded files (theses, documents).

### Source Code (/server/src)
- config/: Configuration modules.
  - config.js: Database configuration.
  - redis.js: Redis client configuration.
- controllers/: Request handlers (Business logic).
  - authController.js: Authentication logic.
  - dashboardController.js: Dashboard data aggregation (e.g., for examiners).
  - defenseEvaluationController.js: Handling defense evaluation submissions.
  - studentInfoController.js: Student data management.
  - documentController.js: File upload and implementation.
  - adminController.js, profileController.js, etc.
- middleware/: Express middleware.
  - authmiddleware.js: JWT verification.
  - rbacMiddleware.js: Role-Based Access Control.
  - upload.js: Multer configuration for file uploads.
- models/: Sequelize database models.
  - defense_evaluations.js: Schema for thesis defense evaluations.
  - studinfo.js, empinfo.js, supervisor.js: User tables.
  - documents_uploads.js, documents_reviews.js: Document tracking.
  - service_requests.js: Student service requests.
  - index.js, init-models.js: Model associations.
- routes/: API route definitions.
  - Maps endpoints (e.g., /api/defense-evaluations) to controllers.
  - defenseEvaluationRoutes.js: Routes for evaluation submissions.
  - dashboardRoutes.js: Routes for dashboard data.
- seeds/: Scripts to populate the database with initial data.
  - seedAdmin.js, createSupervisorUser.js, etc.
- utils/: Helper utility functions.
  - email.js: Email sending implementation.
  - logger.js: Logging utility.
  - authSecurity.js, responseHandler.js.
